# MLOps, 2026

Система эмулирует поступление данных из CSV-файла батчами, проводит их анализ качества, очистку, генерацию признаков, обучает или дообучает модели машинного обучения (случайный лес и нейронная сеть) и сохраняет артефакты (модели, метаданные, предсказания).

## Структура проекта

- `run.py` - точка входа, управление режимами работы.
- `data_streamer.py` - потоковая загрузка и сохранение батчей.
- `data_analyzer.py` - проверка качества, очистка, feature engineering, ассоциативные правила.
- `model_manager.py` - обучение, дообучение, валидация и инференс моделей.
- `config.py` - гиперпараметры и пути.
- Папки: `data/raw`, `data/processed`, `metadata`, `models`.

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Запуск

Программа управляется аргументом `-mode`. Основные режимы:

```bash
# Получение очередного батча, его анализ и обучение/дообучение моделей
python run.py -mode update

# Применение лучшей модели к файлу данных (указать путь)
python run.py -mode inference -file data/new_data.csv

# Вывод сводки по последним обученным моделям
python run.py -mode summary

# Очистка всех артефактов (raw, processed, metadata, models)
python run.py -mode clear

# Только получение и анализ батча без обучения (полезно для проверки)
python run.py -mode get_batch
```

## Результаты выполнения

- **Потоковые батчи** сохраняются в `data/raw/batch_YYYYMMDD_HHMMSS.csv`.
- **Метаданные батча** (период, средняя выплата, распределение целевой переменной) - в `metadata/batch_meta_*.json`.
- **Отчёты о качестве данных** - в `metadata/dq_report_*.json`.
- **Очищенные и обогащённые данные** - в `data/processed/cleaned_batch_*.csv`.
- **Обученные модели** сериализуются в `models/model_<тип>_YYYYMMDD_HHMMSS.pkl`.
- **Симлинки на последние модели** - в `models/latest/`, **на лучшую модель** - в `models/best/`.
- **Метрики обучения** - в `metadata/train_metrics_*.json`.
- **Файл с предсказаниями** - в `data/processed/predictions_HHMMSS.csv`.

## Конфигурация

Основные параметры задаются в `config.py`:
- `SOURCE_PATH` - путь к исходному CSV.
- `BATCH_SIZE` - размер батча.
- `SORT_COLUMN` - колонка для временной сортировки (эмуляция потока).
- `TARGET` - целевая переменная (`CLAIM_PAID_BINARY`).
