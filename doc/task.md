# Описание задачи и архитектуры проекта

## 1. Источник данных и цель моделирования

**Данные:** Файл `data/motor_data11-14lats.csv` - исторические записи о договорах страхования транспортных средств (поля: даты начала/окончания действия полиса, характеристики ТС, пол водителя, премия, страховая сумма, выплаченная сумма убытка и т.д.).

**Целевая переменная:** Бинарный признак `CLAIM_PAID_BINARY` - был ли зафиксирован страховой случай с выплатой > 0.

**Режим работы:** Эмуляция потокового поступления данных путём разбиения исходного файла на батчи, отсортированные по дате начала действия полиса (`INSR_BEGIN`).

## 2. Этапы обработки данных

### 2.1. Потоковая загрузка (DataStreamer)

- Сортировка данных по `INSR_BEGIN`.
- Последовательное извлечение батчей фиксированного размера (`BATCH_SIZE`).
- Состояние (индекс последней прочитанной строки) сохраняется в `metadata/stream_state.json`.
- Для каждого батча создаётся целевая переменная, файл батча сохраняется в `data/raw/`.
- Рассчитываются и сохраняются в JSON метапараметры: количество строк, временной диапазон, средняя выплата, распределение цели.

### 2.2. Анализ качества и очистка (DataAnalyzer)

**Показатели качества данных:**
- **Полнота:** доля заполненных значений по всему датасету и по каждой колонке.
- **Валидность:** проверка отрицательных значений в полях `PREMIUM`, `INSURED_VALUE`, `SEATS_NUM`, `CLAIM_PAID`.
- **Уникальность:** количество полных дубликатов.
- **Ассоциативные правила (Apriori):** выявление устойчивых комбинаций бинарных признаков (например, `SEX=0` → юрлицо и высокая премия) для экспертной проверки корректности данных.

**Очистка:**
- Удаление дубликатов.
- Удаление строк, где более 50% значений пропущено.
- Удаление строк с невалидными (отрицательными) значениями в критичных полях.
- Заполнение пропусков в числовых колонках медианой.

**Feature Engineering:**
- Создание признака `is_legal_entity` (пол = 0).
- Возможно расширение (например, бинаризация категорий по частотности).

Все отчёты о качестве (включая найденные ассоциативные правила и базовую статистику) сохраняются в `metadata/dq_report_*.json`. Очищенный батч записывается в `data/processed/`.

### 2.3. Моделирование (ModelManager)

#### Используемые модели

1. **RandomForestClassifier**
   - Гиперпараметры: `n_estimators=100`, `max_depth=10`, `warm_start=True` (для дообучения), `random_state` из конфига.
2. **MLPClassifier** (нейронная сеть)
   - Гиперпараметры: `hidden_layer_sizes=(32, 16)`, `max_iter=500`, `random_state`.

#### Предобработка признаков

- **Числовые колонки:** импутация медианой + `StandardScaler`.
- **Категориальные колонки:** импутация самым частым значением + `OneHotEncoder` с игнорированием неизвестных категорий.
- Все преобразования объединены в `ColumnTransformer` и помещены в `Pipeline`.

#### Обучение / дообучение

- **При первом запуске** (`models/latest` пуст) модели обучаются с нуля на 80% данных батча, 20% - тест.
- **При последующих запусках** производится дообучение последних сохранённых моделей:
  - **RandomForest:** увеличивается количество деревьев на 25 (`n_estimators += 25`) и модель переобучается на новых данных (с учётом уже построенных деревьев).
  - **MLP:** используется метод `partial_fit` (если доступен).

#### Валидация и контроль качества моделей

- Метрика оценки: **Accuracy** на отложенной выборке.
- Сохраняются метрики в `metadata/train_metrics_*.json`.
- Если модель показала лучший результат среди всех запусков, она помечается как лучшая (симлинк в `models/best/`).
- Все модели версионируются временной меткой, хранятся в `models/`.

#### Инференс

- Для предсказания автоматически выбирается лучшая модель из `models/best/`.
- Результаты (исходные данные + колонка `predict`) сохраняются в `data/processed/predictions_HHMMSS.csv`.

## 3. Хранилище артефактов

- **Сырые данные:** файловая система (`data/raw/`).
- **Обработанные данные:** `data/processed/`.
- **Метаданные:** `metadata/` (JSON-отчёты о качестве, метаданные батчей, метрики обучения).
- **Модели:** сериализация `joblib`, иерархия папок `models/`, `models/latest/`, `models/best/`.
- **Состояние потока:** `metadata/stream_state.json`.
